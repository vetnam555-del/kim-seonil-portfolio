## 1. 수치와 근거 사용 규칙

이 문서는 대화 기록, 고정 프로젝트, 로컬 원본, 자동화 로그를 함께 종합한다. 포트폴리오에 옮길 때는 아래 표시를 기준으로 공개 범위를 결정한다.

| 표시 | 의미 | 사용 기준 |
|---|---|---|
| `[검증 완료]` | 로컬 보고서, QA, 원본 시트, 실행 로그에서 확인 | 공개 포트폴리오에 사용 가능 |
| `[사용자 제공]` | 김선일이 직접 제공한 규모, 기여도, 성과 | 원본 출처를 추가하면 공개 가능 |
| `[대화 기준]` | 광고주 메일, 팀즈 대화, 기존 설명 기준 | 수치를 재확인한 뒤 사용 |
| `[비식별 필수]` | 계정, 매체, 협력사, 정산 정보가 노출될 수 있음 | 브랜드명·계정·거래처·개인정보 제거 |
| `[공개 금지]` | API 키, 비밀번호, 카드·계좌, 고객 개인정보 | 포트폴리오에 사용하지 않음 |

### 현재 사용하지 말아야 할 표현

- 뉴발란스의 근거가 불분명한 `누적 광고 자동 차단 개수`: 지속 사용 가능한 근거가 확인되지 않음.
- 뉴발란스 리포트 작성 시간 `30분 -> 5분` 또는 `60분 -> 5분`: 기준이 충돌하므로 재측정 전까지 정성 표현 사용.
- 강원심층수의 사용자가 제외를 요청한 `월평균 매출 확대` 문구: 사용 금지.
- 대교 하이캠퍼스 `GTM 추적 완전 해결`: 최종 운영 환경 검증이 확정되지 않았으므로 사용 금지.
- 효율 지표를 필요 이상으로 역산해 생성한 매출·전환 수치: 원본 보고서가 없으면 사용 금지.

## 2. 포지셔닝과 핵심 스토리

### 대표 헤드라인

### 서브 카피

검색·쇼핑·디스플레이·SNS 광고를 직접 운영하고, GA4·GTM·매체 API·Google Sheets를 연결해 의사결정과 반복 업무의 정확도를 높여왔습니다. 30여 개 브랜드와 월 최대 15억 원 규모의 운영 경험을 바탕으로, 성과를 보는 관점뿐 아니라 성과가 안정적으로 반복되는 구조까지 설계합니다.

### 포트폴리오 전체 서사

1. 다양한 매체와 큰 예산을 운영한다.
2. 표면적인 CTR·ROAS보다 사업·재고·전환 구조의 문제를 먼저 찾는다.
3. 매체·GA·운영 데이터의 기준을 맞춰 판단 가능한 형태로 만든다.
4. 수기 반복 업무는 자동화하되, 이상 상황에서는 중단되는 검수 장치를 함께 설계한다.
5. 최종적으로 광고주와 팀이 바로 판단할 수 있는 보고·알림·운영 체계로 연결한다.

## 4. 대표 프로젝트 1: 뉴발란스

### 제목

**재고 변동을 매체 운영과 연결한 뉴발란스 전 채널 운영 체계**

### 한 줄 요약

월 10억 원 규모의 인지-전환 매체를 운영하며, 빠르게 변하는 SKU 재고와 쇼핑검색 소재 상태를 연결하는 자동화·QA 체계를 구축했습니다.  
`[운영 규모·기여도 40%: 사용자 제공]`

### 문제 정의

스포츠 브랜드는 사이즈·컬러·시즌별 재고 변동이 크고, 재고가 부족한 상품이 광고에 계속 노출되면 전환 기회 손실과 고객 불편이 발생합니다. 반대로 일시 품절 상품을 너무 오래 OFF하면 수요 회복 기회를 놓칠 수 있습니다. 성인·키즈, 일반 행사·시즌 행사, 개별 상품·소재를 구분하지 않은 수기 점검은 누락과 오판의 위험도 컸습니다.

### 분석 근거

- 신발·의류 상품의 실시간 재고가 소재 ON/OFF 판단과 직결된다.
- 재고 `알 수 없음` 비율이 높은 실행에서 광고를 자동 변경하면 대규모 오조작으로 이어질 수 있다.
- 성인과 키즈 운영은 대상·프로세스·상태 파일을 분리해야 중복 실행과 충돌을 막을 수 있다.
- NAVER·Google·Meta·Kakao·Criteo 등 매체별 지표와 역할이 다르므로 단일 ROAS만으로 예산을 배분하기 어렵다.

### 전략 판단

**재고 임계값을 바탕으로 OFF·재라이브를 자동화하되, 데이터 상태가 건강하지 않은 경우에는 아무 작업도 하지 않는 fail-closed 구조가 필요하다고 판단했습니다.**

### 실행

- 상품 재고, 소재 상태, 수동 OFF 이력을 연결한 쇼핑검색 재고 자동화 파이프라인을 설계했습니다.
- 재고 임계값 기반의 OFF·재라이브 운영 정책을 반영했습니다. 구체 임계값은 시점별 운영 정책에 따라 조정되므로 공개본에서는 범위형으로 표현합니다.
- 성인·키즈 프로필의 프로세스, lock, QA, state, 메일을 분리했습니다.
- `unknown stock`, API 실패, 시트 읽기 실패 등을 감지하면 광고 변경과 시트 업데이트를 중지하도록 했습니다.
- 재고 이상·소진 정체·설정 변경을 구분한 모니터링과 메일 알림 규칙을 설계했습니다.
- 매체 데이터와 GA 데이터를 맞춰 데일리·위클리 코멘트를 생성하고, 누락·지연·범위 차이를 별도 검증했습니다.
- 통합 ROAS만으로 매체를 판단하지 않고, 브랜드검색·신규 유입·리타겟팅·재고 소진 등 매체별 역할을 분리했습니다.

### 결과

- 재고 확인에 일 평균 약 2시간 소요되던 업무를 약 5분 수준으로 줄여 업무 시간을 약 96% 절감했습니다. `[사용자 제공]`
- 오류 시 임의 변경을 막는 건강성 검사를 추가해, 단순 시간 단축이 아니라 운영 안전성까지 높였습니다. `[검증 완료]`
- MOP 기반 쇼핑검색 최적화 사례가 LG CNS MOP 공식 사례로 소개되었습니다. `[공개 링크: https://blog.mop.co.kr/hll]`

### 본인 역할

- 매체별 예산·성과 분석과 운영 의사결정
- 재고 임계값·오류 중단·재라이브 정책 설계
- 성인·키즈 자동화 요건 정의와 QA
- 데일리·위클리 리포트 기준과 광고주 커뮤니케이션
- 기여도 40% `[사용자 제공]`

### 추천 실행 화면·증빙

- 재고 현황 시트와 수동 OFF 이력의 비식별 캡처
- 정상 실행, 중복 방지, 건강성 검사 중단을 비교한 3단 흐름도
- 데일리 코멘트와 위클리 보고서 샘플
- LG CNS MOP 성공 사례 페이지

### 이력서용 3문장

뉴발란스의 월 10억 원 규모 전 채널 매체를 운영하며 매체별 역할과 재고 소진 구조를 함께 관리했습니다. 상품 재고와 쇼핑검색 소재 상태를 연결한 자동 OFF·재라이브 파이프라인과 오류 중단 QA를 구축했습니다. 그 결과 매일 약 2시간 소요되던 재고 확인 업무를 약 5분 수준으로 단축했습니다.

## 5. 대표 프로젝트 2: 강원심층수 천년동안

### 제목

**자사몰·스마트스토어 전환 구조를 재편한 통합 커머스 운영**

### 한 줄 요약

자사몰과 NAVER 스마트스토어의 전환 효율을 비교해 예산·상품·검색 구조를 재편하고, 매체·GA·상품 데이터를 연결한 월간 보고 자동화를 구축했습니다.

### 문제 정의

자사몰·스마트스토어·쿠팡 등 여러 판매 경로로 운영이 분산되어 있었지만 브랜드 인지도가 아직 낮아, 제한된 예산으로 모든 채널을 동시에 키우기 어려웠습니다. 자사몰은 구매 단계의 UI/UX와 결제 편의성에서 이탈이 발생했고, 유료매체 예산이 각 채널의 효율 구간에 도달한 이후에는 매체 밖에서 추가 매출을 만드는 성장 경로도 필요했습니다. 또한 검색·쇼핑·GFA·Meta·Google 데이터와 상품별 성과가 여러 시트에 흩어져 있어 원인 파악에 시간이 필요했습니다.

### 분석 근거

- 2026년 상반기 Payco 제외 기준 광고비 57,564,044원, 구매 완료 4,806건, 전환 매출 136,973,821원, ROAS 약 238%였습니다. `[검증 완료]`
- 전년 대비 광고비 +25.7%, 구매 완료 +82.3%, 전환 매출 +48.6%, ROAS 201% -> 238%로 확인됩니다. `[검증 완료]`
- 5월은 비용 +21.8%에도 CVR -19.5%, 매출 -12.8%, ROAS -28.4%로, 유입보다 전환율 하락이 핵심 문제였습니다. `[검증 완료]`
- 자사몰 CVR는 약 1.5%, 스마트스토어 전환율은 약 3% 수준으로 확인되어 구매 환경의 차이가 전환 효율에 영향을 주는 것으로 판단했습니다. `[사용자 제공·원본 재확인]`
- 생수보다 베이비워터 제품군의 반응과 매출 기여가 높아, 육아 고객군을 별도 성장 세그먼트로 볼 근거가 있었습니다. `[사용자 제공]`
- 상품·검색어·채널별 성과를 분리하면 상품 수요와 낭비 유입을 다른 액션으로 관리할 수 있었습니다.

### 전략 판단

**유료매체 전환은 구매 마찰이 낮은 스마트스토어에 집중하고, 매체별 효율 예산에 도달한 뒤에는 베이비워터와 육아 인플루언서를 연결한 공구·RS 구조로 두 번째 성장축을 만들어야 한다고 판단했습니다.**

### 실행

- 자사몰 CVR와 스마트스토어 전환율을 비교해 스마트스토어 중심의 전환 예산 재배치를 제안했습니다.
- 자사몰·스마트스토어·쿠팡의 역할을 재정의하고, 전환 예산은 스마트스토어에 집중하되 각 매체의 효율 구간까지 예산을 확대했습니다.
- 상품을 A·B·C 등급으로 구분하고, 저효율 상품은 감액·OFF, 브랜드·상품 검색어는 방어, 일반 물 키워드는 탐색 범위를 조정했습니다.

## 6. 대표 프로젝트 3: 대교에듀캠프

### 제목

**CTR 중심의 성과 판단을 CVR·CPA·전환 경로 진단으로 전환**

### 한 줄 요약

소량 노출 소재의 높은 CTR을 성공 기준으로 보던 방식에서 벗어나, 표본 크기·CVR·CPA·랜딩 이탈을 함께 보는 전환 구조 중심의 개선안을 제시했습니다.

### 문제 정의

광고주는 전년 대비 성과가 하락했다고 판단했지만, KPI의 일부가 소진 금액이 적은 고CTR 소재를 기준으로 설정되어 있었습니다. 또한 상담하기 클릭 후 추가 페이지에서 다수의 항목을 입력해야 하는 구조였으므로, 소재 반응보다 전환 경로의 이탈이 병목일 가능성이 있었습니다.

### 분석 근거

- 2025년 비교 기준: CTR 2.54%, CVR 0.66%, CPA 68,869원. `[대화·메일 기준, 원본 재확인 권장]`
- 2026년 5월: CTR 2.25%, CVR 2.24%, CPA 59,169원. `[대화·메일 기준, 원본 재확인 권장]`
- CTR은 소폭 낮아졌지만 CVR은 높아지고 CPA는 개선된 흐름으로, `전년 대비 전체 성과 하락`이라는 단정과 다른 결과였습니다.
- CTR 5% 소재 중 일부는 소진 비용이 약 9천 원으로 표본이 충분하지 않았습니다. `[대화·메일 기준]`

### 전략 판단

**소재 교체만 반복하기보다 소재 반응, 유입, 폼 완료를 각각 분리해 병목을 찾고, 전환 폼의 입력 부담을 낮추는 것이 우선이라고 판단했습니다.**

### 실행

- 소재 판단 기준을 CTR 단일 지표에서 CTR·CPC·CVR·CPA·비용 표본으로 확장했습니다.
- 자사몰 랜딩과 Meta 인스턴트폼 병행 테스트, 이름·연락처 중심의 항목 최소화, 자동 입력 폼을 제안했습니다.
- 인스턴트폼의 DB 양 증가와 상담 품질 하락 가능성을 함께 명시하고, 후속 상담 지표로 품질을 보완하도록 했습니다.
- 매체 로우 데이터, GA 전환, 소재 상태를 분리하고 월이 바뀌어도 기존 보고서 형식을 유지하는 리포트 기준을 설계했습니다.
- 하이캠퍼스 상담 신청의 GTM·GA4 측정 문제에 대해 `신청 시작`-실제 완료 이벤트를 Tag Assistant·DebugView에서 별도 확인하는 QA 기준을 정의했습니다.
- 예산 확인 시 매체 원소진액, 15% 마크업, VAT, 이월분, 추가 요청 분을 분리해 정산 기준을 맞췄습니다.

### 결과

- `성과 하락`이라는 단일 서사를 CTR·CVR·CPA·표본 크기를 함께 보는 구조로 재정의했습니다.
- 현재 병목이 소재만의 문제가 아니라 전환 경로일 가능성을 제시하고, 자사몰·인스턴트폼 병행 테스트안으로 전환했습니다.
- CVR·CPA 비교 수치는 실행으로 새로 만든 결과라기보다, 기존 성과를 올바른 기준으로 재해석한 결과로 표기합니다.

### 본인 역할

- 전년·현재 성과의 동일 기준 비교
- 표본 크기와 전환 퍼널 기반의 성과 진단
- 랜딩·인스턴트폼 병행 테스트와 DB 품질 지표 제안
- 리포트 기준, 예산 정산 로직, GTM 검수 기준 정의
- 기여도 100% `[사용자 제공]`

### 추천 실행 화면·증빙

- 2025·2026 CTR·CVR·CPA 비교 표
- `표본이 작은 고CTR` vs `충분한 표본의 전환 성과` 시각화
- 기존 자사몰 폼과 개선 제안 폼의 단계 비교
- GTM 신청 시작·완료 QA 흐름도

### 이력서용 3문장

대교에듀캠프 캠페인의 전년 대비 성과를 CTR·CVR·CPA·비용 표본으로 재분석했습니다. 소재 반응보다 랜딩·상담 폼의 이탈이 병목일 가능성을 제시하고, 자사몰·Meta 인스턴트폼 병행 테스트와 입력 항목 최소화를 제안했습니다. 매체·GA·소재 데이터의 기준을 맞추고 GTM 전환 이벤트 QA 기준도 함께 설계했습니다.

## 7. 핵심 사례 4-7: 제이에스티나·다이슨·KT알파쇼핑·한샘

- **판단:** 검색 효율을 유지하면서 DA를 더해 신규 수요를 만들고, GA4에서 신규·재구매·채널별 기여를 같은 기준으로 검증해야 한다고 판단했습니다.
- **실행:** SA+DA 통합 미디어믹스, 고객 세그먼트별 메시지, GA4 KPI 재정의, UTM Builder 기반 성과 분석·리포팅 자동화를 설계했습니다.
- **결과:** 대표 운영 구간 GA ROAS 583%, 매체 ROAS 808%, GA 전환매출 약 58.3억 원을 확인했습니다. `[사용자 제공·원본 재확인]`
- **본인 역할:** 전략·운영·측정 구조 설계, 기여도 70%. `[사용자 제공]`

### 다이슨: 콘텐츠 자산을 구매 퍼널로 전환

- **문제:** VAC 콘텐츠와 라이브커머스가 단기 노출에 머물면 브랜드 채널의 지속적인 구매 자산으로 쌓이지 않는 문제가 있었습니다.
- **근거:** 구독자 성장, 콘텐츠별 반응, 라이브커머스 유입과 구매 전환을 함께 보면 콘텐츠가 인지뿐 아니라 구매에 기여하는 구간을 분리할 수 있었습니다.

- **실행:** 콘텐츠 캘린더, 채널별 메시지, 라이브커머스 유입 동선, 성과 리포트를 통합 운영했습니다.

## 8. AX 확장 프로젝트 1: EDIT H 마케팅 뉴스레터

### 제목

**자동 수집보다 잘못된 발행을 막는 데 초점을 맞춘 뉴스레터 운영 파이프라인**

### 문제 정의

매일 10~20개의 마케팅 이슈를 수집·선정·요약하고, 이메일 HTML·웹 아카이브·인스타그램 카드 뉴스로 변환해야 했습니다. 단순 생성 자동화는 복사된 출처, 깨진 HTML entity, 오래된 링크 재사용, 중복 발송의 위험을 키울 수 있었습니다.

### 분석 근거

- 데일리 콘텐츠는 10~20개 이슈, 최소 8개 출처, 단일 출처 35% 이하, reader quality 10점을 품질 기준으로 사용합니다. `[검증 완료]`
- 인스타그램 카드 뉴스는 기본 10장으로 구성하되, `업로드 패키지 생성`과 `실제 게시`를 구분해야 합니다.
- 오류가 있는 HTML과 당일 공개 URL이 준비되지 않은 발행은 성공처리보다 차단이 더 중요했습니다.

### 전략 판단

**생성 속도보다 출처 다양성, HTML 정상성, 발송 증빙, 중복 방지를 먼저 확보하는 fail-closed 출판 구조가 필요하다고 판단했습니다.**

### 실행

- 수집 -> 중복 제거 -> 출처 다양성 검사 -> 요약 -> HTML 생성 -> 유효성 검사 -> 발송 패키지 -> 공개 아카이브 -> 카드 뉴스 순서로 구조화했습니다.
- HTML entity 오류를 사전에 차단하는 validator를 추가하고, 의도적으로 깨진 HTML이 발송 전 차단되는지 테스했습니다.
- 공식 발송과 테스트 발송을 분리하고, lock, 발송 결과, 수신자, 공개 URL, 시각을 증빙하도록 했습니다.
- GitHub Pages의 공개 URL과 로고 자산의 200 응답을 확인한 뒤 발송하는 검수 순서를 두었습니다.

### 결과

- 공식 발송 샘플에서 11개 출처, 23개 후보, reader score 10, 카드 10장, 수신자 16명, 반송 0건을 확인했습니다. `[특정 발행일 샘플, 검증 완료]`
- 실패 발행을 `완료`로 처리하지 않고, 정상 발송·공개·카드 생성 근거가 모두 있을 때만 완료로 판단하는 체계를 만들었습니다.

### 본인 역할

- 데일리·위클리 편집 기준과 출처 다양성 룰 설계
- HTML·이메일·웹·카드 뉴스 제작 파이프라인 구축
- 중복 발송·깨진 HTML·오래된 URL 재사용 방지 QA
- 기여도 100% `[사용자 제공]`

### 이력서용 2문장

매일 10~20개의 마케팅 이슈를 수집·선정·요약하고 이메일 HTML, 웹 아카이브, 인스타그램 카드 뉴스로 변환하는 파이프라인을 구축했습니다. 출처 다양성, HTML entity, 공개 URL, 중복 발송, 수신 결과를 검증해 오류 발행을 사전에 차단했습니다.

## 9. AX 확장 프로젝트 2: 정산·전표 업무

### 제목

**증빙, 계정, 처리 상태를 하나의 검증 흐름으로 연결한 정산 업무 자동화**

### 문제 정의

법인카드 및 매체 정산은 결제 내역, 증빙 파일, 회계 계정, 적요, 저장·상신 상태가 모두 맞아야 합니다. 동일 가맹점의 성공·실패 영수증이 함께 있거나, 당월 기준 파일이 없는 상태에서 작업을 강행하면 잘못된 전표가 생성될 수 있었습니다.

### 분석 근거

- 전월·과거 전표는 가맹점·시간대·금액 패턴에 따라 계정·적요 분류의 기준이 되었습니다.
- 결제 성공·실패 영수증은 파일명만으로 구분하지 않고 금액·시간·상태를 함께 대조해야 했습니다.
- 시스템의 `저장`, `상신`, `취소`, `재등록` 상태는 다른 의미이므로 최종 상태를 다시 읽어 검증해야 했습니다.

### 전략 판단

**정산 자동화의 핵심은 입력 속도가 아니라 `정확한 증빙 -> 과거 근거를 사용한 분류 -> 저장 후 상태 검증 -> 기준 없음 시 중단`의 흐름을 강제하는 것이라고 판단했습니다.**

### 실행

- 결제 내역과 영수증을 금액·시간·성공 여부로 대조했습니다.
- 과거 J-Bill 이력을 기준으로 가맹점·시간대별 계정·적요 분류를 적용했습니다.
- 저장·수정 후 행 개수, 금액, 계정, 첨부, 상태를 재조회했습니다.
- 당월 매체비 기준 파일이 없는 경우 임의 추정으로 계속하지 않고 작업을 중단했습니다.

### 결과

- 6월 J-Bill 28개 항목을 과거 이력과 증빙에 맞춰 저장·수정하고 상태를 검증했습니다. `[특정 월 샘플, 검증 완료]`
- 성공·실패 영수증 중복과 계정 오분류 가능성을 줄이고, 기준 파일이 없을 때 자동으로 중단되는 안전 구조를 확보했습니다.

### 본인 역할

- 증빙·결제·계정·상태 대조 규칙 정의
- 과거 이력 기반 분류 로직과 오류 중단 기준 설계
- 저장 후 시스템 상태 재검증
- 기여도 100% `[사용자 제공]`

### 공개본 비식별 기준

- 회사·거래처·개인·카드·계좌·승인번호 제거
- 실제 영수증 대신 재구성한 흐름도 사용
- `28건` 외의 금액 세부 내역은 비공개

### 이력서용 2문장

법인카드·매체 정산의 결제 내역, 증빙, 회계 계정, 적요, 시스템 상태를 대조하는 자동화 흐름을 구축했습니다. 과거 이력을 기준으로 분류하고 저장 후 상태를 재검증하며, 기준 파일이 없을 때는 임의 처리를 중단하도록 설계했습니다.

- 구조적으로 확인한 결과와 자신이 원인이 되어 만든 결과를 구분한다.

### 정량 결과가 부족한 경우

임의의 성과 수치를 생성하지 않고 아래 4가지 중 하나를 결과로 사용한다.

- 의사결정 개선: 서로 다른 지표를 하나의 판단 기준으로 정리
- 오류 감소: 잘못된 데이터·중복 발송·오조작을 사전 차단
- 사용성 개선: 월·매체 선택으로 자동 갱신, 광고주·팀이 바로 판단
- 운영 표준화: 사람이 바뀌어도 같은 기준으로 실행·검증
